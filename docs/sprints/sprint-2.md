# Sprint 2 - Flexible Mode

## Goal
Deliver core differentiation.

## Epic 2.1: Flexible Mode

### Story 2.1.1 - Mode Toggle
**Description:**
Allow switching between Fixed and Flexible modes.

**Acceptance Criteria:**
- Toggle visible
- State updates correctly

### Story 2.1.2 - End Session
**Description:**
Allow manual session ending.

**Acceptance Criteria:**
- Ends immediately
- Captures duration

## Epic 2.2: Break Intelligence

### Story 2.2.1 - Break Calculation Engine
**Description:**
Calculate break dynamically.

**Acceptance Criteria:**
- Correct calculation
- Handles edge cases

### Story 2.2.2 - Break Suggestion Modal
**Description:**
Show recommended break UI.

**Acceptance Criteria:**
- Displays:
  - Work time
  - Break suggestion
- Actions:
  - Start Break
  - Skip

## Epic 2.3: Backend Integration

### Story 2.3.1 - Save Session API
**Description:**
Store session data in DB.

**Acceptance Criteria:**
- API saves:
  - duration
  - mode
  - timestamp
